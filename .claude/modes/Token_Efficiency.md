# Token Efficiency Mode

## Triggers
Context usage >75%, resource constraints, large-scale operations, --uc or --ultracompressed flag

## Directives
- Use symbol communication for logic, status, technical domains
- Apply context-aware compression for technical terms
- Target 30-50% token reduction while preserving ≥95% information quality
- Use bullet points, tables, concise explanations over verbose paragraphs

## Symbol Systems

**Logic**: → (implies) • ⇒ (transforms) • ← (reverse) • ⇄ (bidirectional) • & (and) • | (or) • : (define) • » (then) • ∴ (therefore) • ∵ (because)

**Status**: [OK] (done) • [FAIL] (failed) • [WARN] (warning) • [ACTIVE] (active) • [PENDING] (pending) • [CRITICAL] (critical)

**Domains**: [PERF] (performance) • [ANALYZE] (analysis) • [CONFIG] (config) • [SECURITY] (security) • [DEPLOY] (deploy) • [DESIGN] (design) • [ARCH] (architecture)

## Abbreviation Systems

**System & Architecture**: `cfg` config • `impl` implementation • `arch` architecture • `perf` performance • `ops` operations • `env` environment

**Development Process**: `req` requirements • `deps` dependencies • `val` validation • `test` testing • `docs` documentation • `std` standards

**Quality & Analysis**: `qual` quality • `sec` security • `err` error • `rec` recovery • `sev` severity • `opt` optimization
